GitHub Actions: Templates vs Marketplace (and why the UI is broken)

This page explains two different things GitHub mixes up in the same place. It also states clearly that GitHub has not fixed the lack of a real community workflow-templates feature. The UI is hard to understand. We use the simplest words we can.

What you want: a full workflow to copy

You want a workflow template. That means: a full YAML file you can copy into your repo. One file. You paste it, maybe change a few lines, and you are done. The file is the template.

What GitHub gives you in the sidebar: actions (not templates)

When you click "Set up a workflow yourself" in the Actions tab, you see a sidebar. That sidebar is the Actions Marketplace. The items there are actions. An action is not a full workflow. It is a single step. You add a step that calls the action and pass it data (inputs). Like calling a function. You do not get a full workflow file to copy. You get a line you add to your file, and you must pass inputs. So the sidebar is not a list of workflow templates. It is a list of building blocks (actions). That is the wrong thing if you wanted "give me a full workflow I can copy."

A lapse in consideration for new technologies: no community workflow templates

GitHub has starter workflows. Those are real templates (full workflow files). But GitHub has stopped accepting new community starter workflows. So the only way to get a full workflow template in front of users is:

  • GitHub’s own small list (curated by GitHub only), or

  • Your organization’s private list (only for repos in that org).

There is no way for the community to publish "full workflow templates" that everyone can see in the same place as GitHub’s. GitHub has neglected to add this for years. There is no sign they will fix it. The Marketplace is for actions (functions can insert to call in your workflow file), not for full workflow templates. Many people assume the sidebar is "more templates" but it is not. It is actions as functions. That lapse is real and it has not been fixed.

Why the UI is confusing

  • The same goal — "I want a workflow" — leads to two different places: starter workflows (full files) and Marketplace (actions as steps). They look similar but are not.

  • "Set up a workflow yourself" sounds like "I will write it from scratch." So users do not expect to find more templates there. They find the Marketplace instead.

  • There is no clear label that says "This is an action (step), not a full workflow template." So people think they are getting a template and end up with a function call and inputs.

More broadly, GitHub’s interface across the site has no clear visual system: no depth, no clear order of importance, just details thrown at you. Here, that makes it hard to tell what a feature is for. GitHub’s design language has been poorly fleshed out since the beginning, and nobody acknowledges that it needs a remake. In fact, many sites call GitHub "user-friendly", which is a joke.

What we do instead: our own workflow templates store

Because GitHub does not offer a real community workflow-templates feature, we run our own store. You can browse full workflow templates there. You get a full YAML file to copy. No "pass data into a function" — you copy the file and use it. The store also shows GitHub’s default workflows and lets developers register their repos as templates. It is built the way a template store should work.

  • In DevCentr: Use the "Workflow templates" entry to open the store in your browser. That store is a separate site (its own domain). It lets you browse and copy full workflow files.

  • Docs for the store: The store has a docs subdomain. It links to DevCentr docs so we do not keep the same text in two places. You read once here.

The store URL and how to submit your repo as a template are in the store site and in the DevCentr app when you open the store.

Domain names and "GitHub"

If you run your own site about GitHub Actions templates, avoid using "GitHub" in your domain name. Using a trademark in a domain can be legally risky and may violate their terms. Use a neutral name (e.g. a name that describes "workflow templates" or "Actions templates" without the word GitHub).

Summary

| What | Meaning | | Workflow template | A full YAML file you copy. One file, done. | | Marketplace action | A step you add. You call it and pass inputs. Not a full workflow. | | GitHub’s sidebar (Set up yourself) | Shows Marketplace actions, not full templates. | | Community workflow templates | GitHub has not added this. We use our own store instead. |
