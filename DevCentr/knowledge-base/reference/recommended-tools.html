Recommended Development Tools

We recommend these tools to improve your dev workflow and maintain consistency across the team.

Platform & Shell

  • Windows Terminal - Official Terminal rewrite from Microsoft.

  • Warp - Modern terminal with integrated AI and superior UX.

  • PowerShell 7 - The modern cross-platform shell.

  • sudo for Windows - Run elevated commands easily.

Package Management

  • UniGetUI - A GUI for winget and other package managers. Use this to install most of the apps below.

  • pnpm - Fast, disk-efficient package manager for Node.js.

Version Control

GUI Clients

  • GitBlade - Clean, modern interface with a focus on speed.

  • GitFiend - A simple, clean, and intuitive Git client.

  • GitAhead - Cross-platform Git GUI with a powerful commit history view.

  • GitExtensions - Feature-rich shell extension and standalone Git UI.

  • git-cola - A powerful, python-based Git GUI.

  • JetBrains GitClient (EAP) - Standalone VCS client from the creators of IntelliJ.

  • Gnome gitg - Disorganized but lightweight. Note: The official wiki lacks screenshots, and several links loop back to the welcome page.

  • GitKraken - Highly polished commercial client. Requires a paid license for professional/private use.

  • Relagit - Experimental client built in Rust.

    • Caveats: Still in early development; lacks submodule support and a proper tree viewer. Performance optimizations are ongoing.

  • SmartGit - Professional client for power users. Paid license required.

  • TortoiseGit - Windows Shell Interface for Git. See Video Demo.

  • wimygit - Features a unique timelapse visualization of repository history.

Spotlight: SourceGit

The SourceGit project is a modern, cross-platform client with an exceptionally active release cycle (Feb 2026 v2026.04).

Technical Considerations:

  • Framework: Built on AvaloniaUI and .NET 10. Requires specific runtimes and may exhibit minor UI glitches (scrollbar overlaps, pixel-perfect rendering) due to its "bleeding-edge" tech stack.

  • Environment: Explicitly requires Git for Windows (does not support MSYS Git).

  • Platforms: macOS users may need to use xattr to bypass security restrictions, though it is now available via signed Homebrew Cask packages.

  • Submodules: Recent versions (v2026.04) have improved automation but users report worktree file statuses may not update correctly after pull/checkout operations.

Workflow & UX Limitations:

  • Built-in Merger: Features a simple merger (added late 2025) but lacks the depth of dedicated 3-way merge tools like Beyond Compare.

  • Config Stability: Recent updates shifted settings like "Show Tags/Submodules as Tree" from global to per-repository, which may be jarring.

  • Performance: Recently optimized for large repos by switching to --pathspec-from-file, but still faces scaling challenges.

Activity Summary:

  • AI Integration: Support for OpenAI and Azure OpenAI.

  • Graphics Support: Fixes for .dds and .tga alpha rendering in diff views.

  • Release Cycle: Follows a "Year.Month" scheme with multiple updates per month.

CLI & Server Tooling

  • GitHub CLI (gh) - Official command-line interface for GitHub management.

  • tea - CLI for Gitea. Useful for managing self-hosted instances. See Self-Hosting Infrastructure for details.

Compilers & Toolchains

  • LLVM / Clang - High-performance compiler toolchain with native Windows compatibility.

    • Usage: winget install LLVM.LLVM -i (Use -i for interactive mode to ensure Clang is added to PATH during installation)

    • Knowledge: See Clang vs. GCC on Windows for why we prefer Clang.

Connectivity & Security

  • OmniSSHAgent - Improved SSH agent management for Windows.

  • OpenSSH-GUI - Graphical interface for managing SSH keys.

Productivity

  • Jam.dev - One-click bug reporting with logs and video.

  • VisBug - Live design debugging in the browser.
