Vercel Deployment

The Antora Themes Gallery is designed for deployment on Vercel.

Prerequisites

  • A Vercel account

  • Your code pushed to GitHub

  • Database credentials ready

Deploying

1. Import Project

  1. Go to Vercel New Project

  2. Click btn:[Import] next to your repository

  3. Configure project settings:

    • Framework Preset: Astro

    • Build Command: pnpm build

    • Output Directory: Leave default

2. Add Environment Variables

Before deploying, add your environment variables:

Variable Value

TURSO_DATABASE_URL

Your Turso database URL

TURSO_AUTH_TOKEN

Your Turso auth token

GITHUB_CLIENT_ID

GitHub OAuth client ID (optional)

GITHUB_CLIENT_SECRET

GitHub OAuth client secret (optional)

GITLAB_CLIENT_ID

GitLab OAuth client ID (optional)

GITLAB_CLIENT_SECRET

GitLab OAuth client secret (optional)

GOOGLE_CLIENT_ID

Google OAuth client ID (optional)

GOOGLE_CLIENT_SECRET

Google OAuth client secret (optional)

3. Deploy

Click btn:[Deploy] and wait for the build to complete.

Custom Domain

Add Domain

  1. Go to your project’s Settings → Domains

  2. Add your domain

  3. Configure DNS as instructed

Update OAuth Callbacks

After adding a custom domain, update your OAuth app callback URLs:

  • https://yourdomain.com/api/auth/callback/github

  • https://yourdomain.com/api/auth/callback/gitlab

  • https://yourdomain.com/api/auth/callback/google

Environment Configuration

Production vs Preview

Vercel supports different environments:

  • Production - Your main branch deployment

  • Preview - Pull request deployments

You can set different environment variables for each.

Sensitive Variables

Mark sensitive variables (tokens, secrets) as "Sensitive" in Vercel to hide their values in the UI.

Build Configuration

The astro.config.mjs is already configured for Vercel:

import vercel from '@astrojs/vercel';

export default defineConfig({
  output: 'server',
  adapter: vercel(),
  // ...
});

Troubleshooting

Build Fails

Check the build logs in Vercel. Common issues:

  • Missing environment variables

  • TypeScript errors

  • Dependency issues

Database Connection Errors

Verify your Turso URL includes the full hostname:

libsql://antora-themes-user.aws-us-east-1.turso.io

OAuth Callback Errors

Ensure your callback URLs in OAuth apps match exactly:

https://your-domain.vercel.app/api/auth/callback/github

Note: No trailing slash, exact case matching.

Local Build Issues on Windows

You may see symlink errors when building locally on Windows:

EPERM: operation not permitted, symlink

This only affects local builds and won’t impact Vercel deployment.

Monitoring

Vercel Analytics

Enable Vercel Analytics for:

  • Page views

  • Web Vitals

  • Real User Monitoring

Function Logs

View serverless function logs in the Vercel dashboard:

  1. Go to your project

  2. Click Logs

  3. Filter by function or status

Updating

When you push to your main branch, Vercel automatically:

  1. Builds the project

  2. Runs any checks

  3. Deploys to production

For controlled releases, use:

  • Branch protection rules

  • Required status checks

  • Preview deployments for testing
